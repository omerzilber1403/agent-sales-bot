# מערכת AI Agent למכירות - הסבר מפורט

## סקירה כללית
אנחנו בונים מערכת AI Agent למכירות שמאפשרת לחברות ליצור בוטים מותאמים אישית למכירות. המערכת כוללת:

### 1. **Backend (FastAPI)**
- **מסד נתונים**: SQLite עם טבלאות לחברות, משתמשים, שיחות והודעות
- **API**: REST endpoints לניהול חברות, משתמשים ושיחות
- **LLM Integration**: חיבור ל-OpenAI GPT לניתוח הודעות ותגובות
- **LangGraph**: מערכת workflow לניהול זרימת השיחה

### 2. **Frontend (React)**
- **דף בחירת חברה**: רשימת חברות קיימות
- **צ'אט אינטראקטיבי**: ממשק שיחה עם הבוט
- **יצירת חברה חדשה**: טופס מקיף להגדרת חברה חדשה
- **ניהול לקוחות**: יצירה ועדכון פרופילי לקוחות

### 3. **מערכת הגרף (LangGraph)**
המערכת עובדת עם 5 nodes עיקריים:

#### **detect_handoff**
- בודק אם ההודעה דורשת העברה לנציג אנושי
- מחפש מילות מפתח כמו "מנהל", "נציג אנושי", "בעיה טכנית"
- **לא מעביר** על שאלות מכירות רגילות (מחיר, זמינות)

#### **match_templates**
- מזהה הודעות סטנדרטיות (שלום, תודה, להתראות)
- יוצר תגובות קצרות ומותאמות אישית
- עובר ל-**friendly_node** אם אין התאמה

#### **llm_fallback**
- נקרא רק אם יש `custom_prompt` לחברה
- משתמש בהוראות מותאמות אישית של החברה
- כולל כל המידע על החברה (מוצרים, ICP, וכו')

#### **friendly_node** ⭐ (הבעיה כאן!)
- נקרא עבור חברות **ללא** `custom_prompt`
- אמור להשתמש במידע החברה מהדאטהבייס
- **הבעיה**: עדיין שואל שאלות עסקיות ל-B2C

#### **handoff_response**
- יוצר תגובה להעברה לנציג אנושי

## הבעיה הנוכחית

### **הבעיה העיקרית**
הבוט של מועדון גלישה (B2C) עדיין שואל שאלות עסקיות כמו:
- "מה התפקיד שלך?"
- "איך העסק שלך עובד?"

### **הסיבה לבעיה**
1. **שגיאת תחביר**: יש ציטוט נוסף בהוראות B2C שגורם לבעיות פרסור
2. **הוראות סותרות**: יש הוראות כלליות שדורסות את ההוראות הספציפיות ל-B2C
3. **זרימת הגרף**: הבוט עובר דרך `friendly_node` אבל לא מקבל את ההוראות הנכונות

### **מה צריך לתקן**
1. **תיקון שגיאת התחביר** בהוראות B2C
2. **חיזוק ההוראות הספציפיות** ל-B2C
3. **הסרת הוראות סותרות** שגורמות לבוט לשאול שאלות עסקיות
4. **וידוא שהבוט מקבל** את המידע הנכון על החברה

## מבנה הנתונים

### **Company Model**
```python
class Company:
    id: int
    name: str
    business_type: str  # "B2B" or "B2C"
    one_line_value: str
    products: List[Dict]
    brand_voice: Dict
    icp: Dict
    custom_prompt: str  # יכול להיות null
    # ... עוד שדות
```

### **User Model**
```python
class User:
    id: int
    company_id: int
    name: str
    occupation: str
    # ... עוד שדות פרופיל
```

### **Conversation & Message Models**
```python
class Conversation:
    id: int
    company_id: int
    user_id: int
    session_id: str

class Message:
    id: int
    conversation_id: int
    role: str  # "user" or "assistant"
    content: str
```

## זרימת הנתונים

1. **Frontend** שולח הודעה ל-`/api/v1/agent/reply`
2. **Backend** מקבל את הנתונים ומכין context
3. **LangGraph** מעבד את ההודעה דרך הגרף
4. **LLM** מקבל system prompt מותאם ומחזיר תגובה
5. **Backend** שומר את ההודעה ומחזיר תגובה ל-Frontend

## הבעיות הספציפיות שצריך לפתור

### 1. **שגיאת תחביר בהוראות B2C**
```python
# בעיה:
business_type_instructions = """
"הוראות ספציפיות ל-B2C (עסק לצרכן) - חובה:

# צריך להיות:
business_type_instructions = """
הוראות ספציפיות ל-B2C (עסק לצרכן) - חובה:
```

### 2. **הוראות סותרות**
יש הוראות כלליות שמזכירות "תפקיד" ו"עסק" שדורסות את ההוראות הספציפיות ל-B2C.

### 3. **וידוא שהבוט מקבל את המידע הנכון**
הבוט צריך לקבל:
- `business_type: "B2C"`
- `company_name: "מועדון גלישה גאולה"`
- `products: [כרטיסיית שיעורי גלישה, שיעור גלישה, השערת גלשן]`
- הוראות ספציפיות ל-B2C
